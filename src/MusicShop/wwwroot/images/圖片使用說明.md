# MusicShop 圖片使用說明

本文件說明如何在 MusicShop 專案中正確放置和使用圖片。

---

## 📁 資料夾結構

```
wwwroot/
└── images/
    ├── banners/      ← 首頁輪播橫幅圖片
    ├── albums/       ← 專輯封面圖片
    └── 圖片使用說明.md
```

---

## 🖼️ 圖片尺寸規格

### 1. 首頁輪播橫幅圖片

**位置**：`wwwroot/images/banners/`

**建議尺寸**：
- **1920 x 600** 像素（推薦）
- **1600 x 500** 像素（替代方案）

**檔案格式**：JPG 或 PNG

**檔案命名範例**：
```
banner1.jpg
banner2.jpg
banner3.jpg
```

**如何使用**：

目前首頁使用色塊，您需要修改 `Views/Home/Index.cshtml` 檔案：

找到這一行：
```html
<div class="carousel-banner" style="background-color: #6f42c1; height: 500px; background-size: cover; background-position: center;">
```

修改成：
```html
<div class="carousel-banner" style="background-image: url('~/images/banners/banner1.jpg'); height: 500px; background-size: cover; background-position: center;">
```

對所有 3 個輪播項目（banner1.jpg、banner2.jpg、banner3.jpg）重複此操作。

---

### 2. 專輯封面圖片

**位置**：`wwwroot/images/albums/`

**建議尺寸**：
- **500 x 500** 像素（推薦）
- **800 x 800** 像素（高解析度）
- **必須是正方形**

**檔案格式**：JPG 或 PNG

**檔案命名範例**：
```
bts_proof.jpg
blackpink_born_pink.jpg
twice_between_1and2.jpg
album_001.jpg
```

**如何使用**：

在資料庫的 `Albums` 資料表中，`CoverImageUrl` 欄位有兩種儲存方式：

#### 方式 1：只儲存檔案名稱（建議）

資料庫欄位值：
```
bts_proof.jpg
```

View 中使用：
```html
<img src="~/images/albums/@album.CoverImageUrl" alt="@album.Title">
```

#### 方式 2：儲存完整相對路徑

資料庫欄位值：
```
~/images/albums/bts_proof.jpg
```

View 中使用：
```html
<img src="@album.CoverImageUrl" alt="@album.Title">
```

#### 方式 3：儲存完整 URL（如果使用 CDN）

資料庫欄位值：
```
https://cdn.example.com/albums/bts_proof.jpg
```

View 中使用：
```html
<img src="@album.CoverImageUrl" alt="@album.Title">
```

## 💡 快速測試步驟

### 測試輪播橫幅圖片

1. 準備 3 張圖片（1920x600 或 1600x500）
2. 命名為 `banner1.jpg`、`banner2.jpg`、`banner3.jpg`
3. 放入 `wwwroot/images/banners/` 資料夾
4. 修改 `Views/Home/Index.cshtml` 中的 3 個輪播項目（參考上方說明）
5. 執行 `dotnet run` 並開啟瀏覽器查看

### 測試專輯封面圖片

1. 準備幾張正方形圖片（500x500 或 800x800）
2. 命名為 `album1.jpg`、`album2.jpg` 等
3. 放入 `wwwroot/images/albums/` 資料夾
4. 使用 SQL 更新資料庫：

```sql
-- 方式 1：只儲存檔案名稱
UPDATE Albums SET CoverImageUrl = 'album1.jpg' WHERE Id = 1;
UPDATE Albums SET CoverImageUrl = 'album2.jpg' WHERE Id = 2;

-- 方式 2：儲存相對路徑
UPDATE Albums SET CoverImageUrl = '~/images/albums/album1.jpg' WHERE Id = 1;
```

5. 執行 `dotnet run` 並開啟瀏覽器查看

## 🔧 圖片優化建議

### 檔案大小優化

- 輪播橫幅：控制在 200KB 以內
- 專輯封面：控制在 100KB 以內

### 推薦工具

- **TinyPNG**：https://tinypng.com/ （線上壓縮）
- **ImageOptim**：https://imageoptim.com/ （Mac 應用程式）
- **Squoosh**：https://squoosh.app/ （Google 線上工具）

### 命名規則

- 使用英文、數字、底線
- 避免空格和特殊字元
- 小寫字母（建議）
- 範例：`bts_proof_album.jpg`、`banner_sale_2024.jpg`

## ❓ 常見問題

### Q1：圖片不顯示怎麼辦？

檢查以下項目：
1. 圖片檔案是否放在正確的資料夾（`wwwroot/images/albums/` 或 `wwwroot/images/banners/`）
2. 檔案名稱是否與資料庫中的 `CoverImageUrl` 欄位一致
3. 檔案名稱是否包含空格或特殊字元
4. 瀏覽器開發者工具（F12）中是否有 404 錯誤

### Q2：如何在沒有圖片時顯示預設圖片？

目前首頁已經包含預設處理：
- 當 `CoverImageUrl` 為空或 null 時，會顯示漂亮的漸層背景和音符圖示

### Q3：可以使用外部 CDN 的圖片嗎？

可以！只要在資料庫的 `CoverImageUrl` 欄位填入完整的 URL：
```
https://cdn.example.com/albums/bts_proof.jpg
```

### Q4：如何批次上傳圖片？

目前專案沒有圖片上傳功能，建議：
1. 手動將圖片複製到 `wwwroot/images/albums/` 資料夾
2. 使用 SQL 批次更新資料庫的 `CoverImageUrl` 欄位
3. 未來可以在後台管理功能中加入圖片上傳功能
